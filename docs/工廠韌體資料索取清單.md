# 工廠韌體資料索取清單（SIP-Player-2024 / 板 v6.1.2）

> 目的：取得對「板 v6.1.2 + GOKE SoC」這版設備做**韌體優化/改寫 + 系統適配**所需的開發資料。
> 已有：`websetsip.c/.h`、`main.c`（REST `:80` 設定服務源碼）。缺的見下。
> 依「能做到的深度」分三組，工廠能給到哪組，就決定我們能走 T1 還是 T2。

---

## A. 先確認（小而關鍵，一句話就能回）

1. **SoC 確切型號**（GOKE GK????）、CPU 架構/核心數/主頻
2. **RAM / Flash 容量**與分割表（從 `/system/info` 反推可用 RAM 僅約 30MB，請確認實體容量）
3. **DBP 發現協定的監聽埠號**與傳輸（TCP？格式？）—— 直接給就不必抓包
4. 設備 **root 帳密** 與 **SSH 登入方式**（實測 `:9521` 跑 OpenSSH 5.5）
5. `websetsip` 與 `/opt/termapp` 的**啟動方式**（init script / inittab / 服務管理）
6. rootfs 是否可寫、如何**替換 `/opt` 下的二進位**（OTA？手動？）

## B. 解鎖 T1 — 重編 REST 層 + 加整合 agent（中等門檻，低風險）

7. **交叉編譯工具鏈**（GOKE BSP/SDK 的 toolchain）：gcc 版本、sysroot、ABI（arm uclibc / musl？軟浮點或硬浮點）
8. `websetsip` 依賴的**框架原始碼或 header + 靜態庫**：
   `event` / `socketbase` / `aio_audio` / `keyvaluefile` / `hostsid` / `webapi` / `readtemp`
   （`cjson` 是公開的，可略）
9. 該模組的 **Makefile / 建置腳本**
10. 設備上跑自製二進位的限制（記憶體上限、可用的 libc 函式、是否有看門狗會殺非預期進程）

## C. 解鎖 T2 — 全韌體改寫/優化（高門檻，需救磚安全網）

11. **完整 BSP**：kernel 源碼 + 設定檔、bootloader、rootfs build system（buildroot/yocto/自製）
12. **`/opt/termapp` 完整源碼**：SIP 堆疊、RTP/音訊管線、**DBP 發現協定**、`/tmp/sip.sdk` IPC 指令集
13. **音訊 HAL（`aio_audio`）規格** + DAC/ADC/功放 codec 晶片型號
14. **燒錄方式** + **救磚/恢復流程**：UART console（腳位/鮑率）、TFTP/SD 恢復、完整韌體映像
15. **板子原理圖 + BOM**（v6.1.2）

## 為什麼要這些（給工廠的說明）

我們要把這版設備**深度接入既有的 SIP 廣播管理系統**（Asterisk + gateway），可能新增：與 gateway 的設定下發/狀態回報通道、多播廣播編組、降低輪詢負載等。
- 若只要在 REST 層擴充 + 加一支整合 agent → 需要 **A + B**。
- 若要改音訊管線/SIP 行為/DBP 等核心 → 需要 **A + B + C**（並務必提供救磚流程，避免變磚）。
