# GT-SIP-GW 設備管理網頁使用手冊 — IO 觸點與組播監聽區

> 適用對象：現場工程師／校園系統管理員
> 適用版本：device-web 2026-07-18 版（含「IO 觸點」「📡 組播監聽區」兩頁）；組播監聽區與 IO 功能需搭配支援對應端點的新韌體（工廠需求單交付中），**舊韌體下兩頁會顯示「此韌體未回報…或尚未支援此功能」屬正常現象**，不影響其他頁面。
> 版本紀錄：2026-07-18 初版（經瀏覽器實測全流程後撰寫）。

## 1. 登入與安全

1. 瀏覽器開設備位址（例 `http://<設備IP>/`）。同網段直連；跨網段請確認路由可達。
2. 預設帳號 `admin`。**以預設密碼登入會被強制先改密碼**（至少 8 字元、含英文與數字）——改完會登出，用新密碼重新登入。
3. 頁首「已連線」燈號＝與設備通訊正常。HTTP 明文連線會顯示黃色安全提醒——正式環境建議啟用 HTTPS（韌體支援後）。
4. 閒置會自動登出；登入連續失敗會暫時鎖定（倒數提示）。

## 2. 組播監聽區（📡）

### 2.1 這頁在做什麼

設定這台喇叭/閘道器**同時監聽的最多 16 個組播區（zone）**。每區一個組播位址:埠；多區同時來音訊時，**依優先權播放單一流、不混音**——高優先權即時蓋掉低優先權，高優先權結束後自動恢復低優先權。

**校園規劃慣例**（與平台側一致）：

| Zone | 用途 | 優先權 |
|---|---|---|
| 1 | 緊急廣播 | 1（最高） |
| 2 | 全校鐘聲/廣播 | 2 |
| 3 | 年級廣播 | 3 |
| 4 | 大樓/區域廣播 | 4 |
| 5+ | 班級/其他 | 依需求 |

> Zone 1 與「SIP / 組播」頁的「組播接收」卡片是**同一份設定**（該卡片＝zone 1 快速設定），兩處改任一處皆可。

### 2.2 欄位規則

- **組播位址**：`224.0.0.0`–`239.255.255.255`（校園慣用 `239.x.x.x`）。
- **組播埠**：1024–65535。
- **優先權**：1–16，數字越小越優先；**所有已啟用區之間不得重複**——重複時列旁即時出現「⚠ 優先權重複」且無法儲存。
- **音頻編碼**：G.722（寬頻，喇叭韌體 v2.1.1+）或 G.711U。需與平台側該組播域設定一致。
- **啟用此監聽區**：勾了才會實際監聽。

### 2.3 儲存行為（重要）

- 「**儲存全部監聽區**」＝**16 區整表一次送出**；任一列驗證不過（會標明是哪個 Zone）整批都不會儲存。
- **空白列**（位址空、埠/優先權空）＝尚未配置的佔位列，不會被檢查、照常允許儲存。
- **半成品列會被擋**：只填了埠或優先權、位址卻空白的列（不論是否啟用）會報錯——請填完整或整列清空。
- 停用（取消勾選）一個已配置的區＝**保留設定值待命**，不會清空；要清掉請把該列欄位清空再存。
- 儲存成功顯示「已儲存」，設定**即時生效免重啟**。

### 2.4 常見問題

| 現象 | 原因／處理 |
|---|---|
| 「此韌體未回報多監聽區設定，或尚未支援此功能」 | 韌體尚未支援多監聽區（舊版）。升級到支援多區的韌體後此頁自動生效。單區監聽請用「SIP / 組播」頁。 |
| 「⚠ 優先權重複」 | 兩個已啟用區用了同一優先權數字，改成不重複即可（停用區的優先權不算）。 |
| 「Zone N 組播位址錯誤」 | 位址不在 224–239 開頭的組播範圍，或格式錯誤。 |
| 存了但沒聲音 | 確認平台側該組播域的位址:埠與此處一致、該區已啟用、喇叭音量正常；跨網段需網路設備放行 IGMP/組播路由。 |

## 3. IO 觸點

設定板上 5 路乾接點（IO1–IO5）的功能綁定：每路可設**方向**（輸入/輸出/停用）、**觸點型態**（NO 常開／NC 常閉）、**觸發方式**與**綁定動作**（組播 PTT、撥打預設號碼、掛斷、SOS…）。

- 「動作」下拉顯示「（請選擇）」時代表尚未綁定——必須明確選一個動作才能存（防止誤存預設值覆蓋現場設定）。
- 要停用某路 IO：把「方向」設為停用（不是把動作留空）。
- 輸入/輸出方向與動作不相容時會擋存（例：輸出腳綁輸入型動作）。
- 舊韌體顯示「此韌體未回報 IO 設定」＝該韌體尚未支援，升級後自動生效。
