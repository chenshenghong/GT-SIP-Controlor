# 韌體整合說明 — 讓設備 :80 提供內建管理網頁

> 對象：設備韌體工廠。目標：在現有 `websetsip.c` 補上唯一缺口——讓 `GET /` 回傳內建管理網頁。
> 已驗證：本包產生的回應，body 解壓後與 `device-web/index.html` **位元組完全相同**；handler 以 `cc -Wall` 編譯零警告。

## 背景（一句話）

原 `http_callback()` 只 dispatch 19 條 JSON 路由，**沒有任何靜態檔/HTML 處理**，所以 `GET /` 不會回網頁。本包只補這一條路由，**不動原本 19 條 API**。

## 檔案清單

| 檔案 | 用途 |
|---|---|
| `make-embed.sh` | 把 `../index.html` 壓成 gzip 並產生 `web_index_gz.h`（可重現）。網頁更新時重跑。 |
| `web_index_gz.h` | 已產生好的 C 標頭：`web_index_gz[]` + `web_index_gz_len`（gzip 後約 15 KB）。 |
| `serve_index.c` | `request_get_index()` handler + 路由插入位置（含註解）。 |

## 整合三步驟

1. **放入內嵌資料**：把 `web_index_gz.h` 複製到韌體原始碼樹（與 `websetsip.c` 同目錄即可）。

2. **加入 handler**：把 `serve_index.c` 裡的 `#include "web_index_gz.h"` 與 `request_get_index()` 函式，貼進 `websetsip.c`（建議放在其他 `request_*` handler 附近）。函式沿用既有的 `HBI_WEB_SERVER`、`web_snd_data`、`memcpy/malloc`，**無新增相依**。

3. **插入路由（順序很關鍵）**：在 `http_callback()` 的 `if (len > 0) {` 之後、**第一個 `if (strncmp("/auth/login", ...))` 之前**，貼入：

   ```c
   if (len == 1 && url[0] == '/')                          /* GET /           */
   {
       request_get_index(client);
   }
   else
   if (len >= 11 && strncmp("/index.html", url, 11) == 0)  /* GET /index.html */
   {
       request_get_index(client);
   }
   else
   if (strncmp("/auth/login", url, len) == 0)              /* ←原本的鏈接這裡 */
   ...
   ```

   > ⚠ **務必放最前面**：既有比對 `strncmp(route, url, len)` 是 prefix 比對，`GET /`（`len==1`）會誤命中 `/auth/login`。用「精確長度」在鏈最前面攔截才正確。

編譯、燒錄，瀏覽器開 `http://<設備IP>/` 即見登入頁。

## 注意事項

- **網頁存 UTF-8**：`index.html` 是 UTF-8（與韌體既有 JSON 的 GBK 不同）。handler 已回 `Content-Type: text/html; charset=UTF-8`，請勿把它轉成 GBK。
- **body 是二進位**：gzip 內含 `0x00`，複製 body 只能用 `web_index_gz_len`（已處理），**切勿** `strlen`/`snprintf` 處理 body。
- **gzip 相容性**：所有現代瀏覽器都支援 `Content-Encoding: gzip`，可無條件回 gzip。
- **更新網頁**：日後 `index.html` 有改，重跑 `sh make-embed.sh` 覆蓋 `web_index_gz.h`、重新編譯即可。

## 替代做法（若貴廠偏好讀檔而非編譯內嵌）

把 `index.html`（可先 gzip）放到設備 flash 固定路徑（如 `/etc/sip_web/`），handler 改成開檔讀出再送。內嵌（本包預設）的好處是與韌體映像原子綁定、無檔案系統相依、較不易出錯，故建議優先採用。

## 驗收

```
# 設備上線後，於同網段執行：
curl -s -D - http://<設備IP>/ -o /tmp/page.gz        # 看標頭應有 Content-Encoding: gzip
gunzip -c /tmp/page.gz | head -1                      # 應為 <!DOCTYPE html>
```
瀏覽器直接開 `http://<設備IP>/` 應顯示「GT-SIP-GW · 廣田資訊」登入頁，登入後可操作儀表板與各設定頁。
